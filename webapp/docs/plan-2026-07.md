## 1️⃣ 止损纪律硬化 · `feat/stop-discipline`

**目标**:让「每一笔都有止损、单笔风险受控」从提示变成默认行为。本次 review 已加了两处提示(风险卡无止损角标、计算器追高警告),这里补完。

**要做**
- 持仓页顶部:有无止损持仓时,显示醒目红色 banner「⚠ N 个持仓未设止损,组合风险未封顶」,点击定位到相应行。
- 添加/记入持仓:止损为**必填**;确需无止损(如短期套利)时二次确认并打标,风险统计里单列。
- 行内编辑:保存时若把止损清空 → 弹确认。
- 设置页新增两个阈值:**单笔最大风险 %**(默认 1.5%)、**组合最大热度 %**(默认 6%);计算器/持仓页超阈值标红。
- (可选)组合风险敞口卡片:把「已计入止损的风险」与「无止损持仓的成本敞口」分两行显示,不再用一个会误导的数字。

**数据模型**:复用现有表;`settings` 加 `maxRiskPct` / `maxHeatPct`;`positions` 加布尔标记 `no_stop_ack`(默认0)。

**验收**:任意新建持仓不填止损会被拦;无止损时首页有红 banner;超阈值有红字。

## 2️⃣ 结构化交易日志 + 绩效面板 · `feat/trade-journal`

这是**最直接服务「提升」**的一块。分三部分:每笔交易日志(A)、每日市场环境日志(B)、绩效面板(C)。

### A. 每笔交易日志(结构化)
下单/平仓时或事后补填,关联到 `positions`。

| 字段 | 类型 | 说明 |
|---|---|---|
| setup 类型 | 单选 | VCP突破 / 杯柄 / 平台整理 / 高紧密旗形(HTF) / 均线回踩 / Power Play / 反转日 / 其他 |
| 入场理由 | 文本 | 为什么现在买(枢轴、量、大盘配合…) |
| 信念度 | 1–5 | 对这笔的把握 |
| 情绪 | 1–5 | 下单时的情绪(FOMO?犹豫?) |
| 退出理由 | 单选+文本 | 触止损 / 到目标减仓 / 跌破20MA / 大盘转弱 / 情绪化割肉 / 其他 |
| 错误标签 | 多选 | 追高、止损过宽、仓位过重、无视大盘、过早离场、死扛… |
| 教训 | 文本 | 这笔最大的一条心得 |

> 纪律自评自动算(非手填):是否有止损、单笔风险是否 <阈值、买点是否在枢轴附近(与 #3 联动)。

### B. 每日市场环境日志(自动填 + 手填)
扩展现有「每日复盘」,把自由文字升级为结构化环境卡。

**系统自动带入**(当天盘后锁定,复用 `/api/market`、VIX、情绪指数、`/api/heatmap/sectors`、持仓盈亏):
- 标普/纳指/道指 涨跌、VIX、Fear&Greed 情绪值(现在环境条已有「情绪77」)
- 领涨/领跌板块 Top3、板块热力快照
- 我的当日组合:市值、浮盈亏、热度、仓位占比

**需要你判断、手填的**(留结构化输入框,不是一大段自由文字):
- 大盘研判:Follow-Through Day?Distribution Day 计数?趋势=多/空/震荡
- 美联储发声 / 利率预期变化
- 重大宏观数据:CPI / PPI / 非农 / FOMC 结果 + 你的解读
- 国际局势 / 突发事件
- **我的感受 / 情绪分 / 今日纪律打分**

> 落地:新表 `market_journal(date PK, auto_json, fed_note, macro_events, intl_note, feeling, discipline_score, trend_view, ...)`;盘后 job 自动写 `auto_json`,手填字段随时可改。宏观事件可做一个小日历(标 CPI/FOMC 日),到点提示你填。

### C. 绩效面板(Minervini 最看重的数字)
基于 `trades` + `journal` 自动统计,页面顶部大卡 + 明细表:

| 指标 | 为什么重要(Minervini) |
|---|---|
| 胜率(batting average) | 基础 |
| 平均盈利% / 平均亏损% | 核心 |
| 盈亏比(avg gain ÷ avg loss) | 目标 ≥2:1 |
| **期望值** = 胜率×均盈 − 败率×均亏 | 一笔的数学期望,判断策略是否正期望 |
| 最大回撤 | 风控 |
| 盈利单/亏损单**平均持有天数** | 检验「赢家持有久、输家砍得快」 |
| **按 setup 类型分组** 的胜率/期望值 | 找出哪类 setup 最赚钱,加码它、淘汰差的 |
| 纪律达成率 | 有多少笔真正按规则执行 |

**数据模型**:新表 `journal`(见上 A 的字段 + `position_id`、`ticker`、`created_at`);绩效全部由 `trades`/`positions`/`journal` 实时聚合,不额外存。

**验收**:平掉一笔后能在绩效面板看到它计入胜率/期望值;能按 setup 类型看到分组统计;每日复盘能看到自动带入的市场环境 + 我的手填。

## 3️⃣ 枢轴跟踪池 + 买点识别 · `feat/pivot-tracker`

**定位**:筛选(全市场是否满足 SEPA)交给 **TradingView**;把不超过 ~100 只的候选丢进本系统,系统**每天盘后**跟踪它们是否**接近或突破枢轴点**,并提醒你。

### 数据模型
新表 `tracker`:

| 字段 | 说明 |
|---|---|
| ticker | 代码 |
| pivot_price | 枢轴价(系统建议 → 你确认/改) |
| pivot_source | auto / confirmed / manual |
| buy_limit_pct | 可买区上限(默认枢轴上方 5%,超了=追高) |
| stop_ref | 参考止损(默认枢轴下方 7–8% 或近期低点) |
| base_stats_json | 基座统计(见下,信息性) |
| status | watching / approaching / triggered / bought / expired |
| note, added_at, updated_at | |

### 枢轴自动建议算法(VCP-lite,建议→你确认)
用已有 `bars` 日线缓存,回看约 3–6 个月:
1. 识别最近一段**整理基座**(前段有上涨,随后在某阻力下方横盘)。
2. **枢轴建议 = 基座最近的摆动高点/阻力**(股票要突破的那条线)。
3. 计算并展示基座质量(帮你判断要不要确认):
   - 基座长度(周)、深度(%)、**收缩次数**(左到右回调是否一次比一次浅,如 25%→15%→8% = 典型 VCP)
   - **缩量**程度(近期均量 vs 50 日均量)
   - 当前**距枢轴 %**、是否仍在 Stage 2
4. 你在跟踪池里一键**确认或改枢轴价** → `pivot_source=confirmed`。

> 说明:自动识别只做**建议**,最终以你确认为准(这正是你选的方案),避免机器误判 VCP 带来的坏信号。

### 每日盘后跟踪(复用现有 scheduler)
每个交易日收盘后,对池内每只票计算并判定状态:
- 距枢轴 %(现价 vs 枢轴)
- **接近**:进入枢轴下方 X%(默认 3%)内 → `approaching`
- **突破**:现价 ≥ 枢轴 → `triggered`;并判断**是否放量**(当日量 > 1.5× 50 日均量)
- **假突破警惕**:突破但缩量 → 单独提示
- **已过可买区**:现价 > 枢轴×(1+buy_limit_pct) → 提示「追高勿追」
- Stage 2 是否仍成立(跌破则降级/移出提示)

### 提醒(Telegram + 站内 banner,去重复用 `notify_state`)
每类状态变化推一次:接近枢轴、放量突破(可买)、缩量突破(存疑)、已过可买区、跌破基座出局。文案带:代码、枢轴、现价、距枢轴%、量能、建议动作。

### UI
- 新页/新区「枢轴跟踪」:表格列出池内票(距枢轴、状态、Stage、量能、基座质量、建议动作、确认枢轴按钮);支持从 TradingView 批量粘代码导入。
- 个股看板 K 线**叠加枢轴线**(lightweight-charts price line)+ 可买区带,一眼看清位置。
- 点击跟踪池某票 → 跳到个股看板并高亮枢轴。

**验收**:粘入一批代码 → 系统给出建议枢轴 + 基座质量 → 我确认后,某票盘后接近/放量突破时,Telegram 和站内都收到提醒;K 线上能看到枢轴线。
